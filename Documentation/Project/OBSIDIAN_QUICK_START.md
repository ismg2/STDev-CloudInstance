# Obsidian Option B Tutorial

This tutorial sets up a practical Requirement to Release cockpit in your Obsidian vault using a folder architecture and plugin-driven dashboards.

This is the minimal version to avoid complexity.

## What you get
- Structured workflow from requirement to release
- Repeatable Codex discussion and decision routine
- Task tracking and automatic views
- Minimal files only

## Daily operating loop
1. Create requirement note from template.
2. Run Codex prompt 1 and 2 (debate and decision).
3. Mark TODO tasks in the requirement note.
4. Implement in code branch and run regression.
5. Update release note and close requirement.
